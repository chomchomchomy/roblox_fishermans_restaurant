# Continuous Learning Skill

「一度解決した問題は、二度と考えない」
このプロジェクト独自の知見を資産化し、私のパフォーマンスを永続的に向上させます。

## 学習の対象 (What to learn)
- **バグと解決策**: 「〇〇エラーが出た時は、△△の設定を変える」といった具体的な回避策。
- **プロジェクトの規約**: 「変数名は〇〇形式にする」「テストファイルはここに置く」などの暗黙の了解。
- **外部仕様の癖**: 「このサイトをスクレイピングする際は、User-Agentの設定が必須」などの外部要因。
- **ワークフローの改善点**: 「Verification Loopの前に、まずこのスクリプトを走らせる方が効率的」などの手順の最適化。

## 実行ワークフロー

### ステップ 1: Harvest (収穫)
セッションの最後、または大きなマイルストーンを完了した時に、私は自問自答します。
「今回、このプロジェクト固有で学んだことは何か？次に私が作業する時に、何を知っていればもっと早く終わるか？」

### ステップ 2: Record (記録)
学んだ内容を、プロジェクト直下の `.agent/knowledge/learned_patterns.md` に追記します。
書式は以下を推奨：
- **[日付] カテゴリ**: 事象の概要
- **Cause**: 原因
- **Insight**: 得られた知見・解決策

### ステップ 3: Recall (回想)
新しいセッションを開始する際、私は `Iterative Retrieval` スキルを活用して、まず `.agent/knowledge/` 下の学習記録を確認します。これにより、前回のコンテキストがリセットされていても、「前回の続き」として賢く動けます。

## 私が意識すること
私は「ただ言われたことをやる」だけでなく、「このプロジェクトにとって何が資産になるか」を常に考えます。作業が終わった時に、「今回の学びを learned_patterns.md に記録しておきました」とあなたに報告します。
